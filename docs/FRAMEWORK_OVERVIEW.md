## Introduction

The Strategic Pyramid Framework is a comprehensive approach to strategy development that bridges the gap between aspiration and execution, between planning and adaptation. Unlike traditional strategy approaches that produce static documents, this framework creates a **living strategy ecosystem** that evolves with your reality.

### What Makes This Different

Most strategy frameworks fall into one of two camps:
- **Too abstract**: Beautiful vision statements with no execution path
- **Too tactical**: Detailed project plans with no strategic coherence

The Strategic Pyramid Framework does both:
- **Grounds strategy in reality** through explicit context analysis
- **Forces hard choices** through structured prioritization
- **Maintains coherence** through red thread traceability
- **Enables adaptation** through continuous learning loops

## The Problem We're Solving

### The Traditional Strategy Failure Pattern

```
Week 1-4:  Leadership offsite → Beautiful strategy deck created
Week 5-12: Strategy "communicated" → Slide decks distributed
Month 3-6: Teams confused → "What does this mean for us?"
Month 6+:  Strategy forgotten → Deck in drawer, business as usual
Year end:  Repeat cycle → "Let's refresh our strategy"
```

### Why Strategies Fail

Based on extensive research and practical experience, strategies fail because:

1. **Disconnection from reality**: Strategy created in isolation from context
2. **No forced choices**: Everything is "strategic" so nothing is prioritized
3. **Missing red thread**: No clear line from vision to individual action
4. **Static documents**: No mechanism for learning and adaptation
5. **Poor transparency**: Different people see different pictures
6. **Execution void**: Gap between "what we'll do" and "who does what"

### What Success Looks Like

A successful strategy:
- ✅ Everyone understands WHY we're doing this
- ✅ Leadership has made HARD CHOICES about focus
- ✅ Clear RED THREAD from purpose to individual contributions
- ✅ Regular LEARNING LOOPS that inform adjustments
- ✅ TRANSPARENT to all stakeholders
- ✅ LIVING DOCUMENT that evolves with reality

## Core Philosophy

### Foundation Principles

#### 1. Context Before Commitment
**Strategy without context is hope, not strategy.**

Every strategy must start with explicit understanding of:
- Internal reality (strengths, constraints, capabilities)
- External reality (opportunities, threats, market dynamics)
- Stakeholder landscape (who matters, what they need)
- Strategic tensions (trade-offs we must navigate)

#### 2. Coherence Through Connection
**Every element must trace to vision and forward to action.**

The "red thread" principle:
- Every strategic driver links to purpose
- Every intent links to drivers
- Every commitment links to intents
- Every team objective links to commitments
- Every individual contribution links to team objectives

This creates **vertical alignment** and prevents orphaned initiatives.

#### 3. Choices Over Wishes
**Strategy is as much about what we DON'T do as what we do.**

Real strategy requires:
- Limiting strategic drivers to 3-5 (more = unfocused)
- Assigning primary ownership (no shared accountability)
- Explicit trade-offs (what are we sacrificing?)
- Horizon planning (what now vs. later vs. future)

#### 4. Living Over Static
**Strategy is not a document, it's a practice.**

Strategy must:
- Adapt based on learning
- Respond to context shifts
- Evolve as we execute
- Incorporate feedback continuously

This is where most frameworks fail - they stop at planning.

### Philosophical Influences

Our framework draws from:

- **Golden Circle (Simon Sinek)**: Start with WHY, then HOW, then WHAT
- **3 Horizons Model (McKinsey)**: Manage present while building future
- **Strategyzer Approach (Osterwalder)**: Visual, testable, iterative
- **Agile/Lean Principles**: Inspect and adapt, fast feedback loops
- **Playing to Win (Lafley/Martin)**: Strategy as integrated set of choices
- **Team Dynamics (Lencioni)**: Trust, conflict, commitment, accountability
- **Systems Thinking**: Understanding interconnections and feedback loops

## The Three-Step Framework

### Overview

```
┌─────────────────────────────────────────────────────────────┐
│  STEP 1: UNDERSTAND (Context & Discovery)                   │
│  ↓                                                           │
│  Ground strategy in reality                                 │
│  Build shared understanding                                 │
│  Surface tensions and opportunities                         │
│  Output: Context document + Prioritized opportunities       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 2: DEFINE (Strategy & Plan)                           │
│  ↓                                                           │
│  Make clear choices                                         │
│  Build coherent pyramid (9 tiers)                           │
│  Create red thread traceability                             │
│  Output: Complete strategy with execution plan              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 3: ADAPT (Living Execution)                           │
│  ↓                                                           │
│  Monitor progress and context shifts                        │
│  Learn from execution                                       │
│  Adjust based on evidence                                   │
│  Output: Evolving strategy + Learning journal               │
└─────────────────────────────────────────────────────────────┘
```

### Step 1: UNDERSTAND (Context & Discovery)

**Duration**: 1-2 weeks (lightweight) to 4-6 weeks (comprehensive)

**Purpose**: Ensure strategy is grounded in reality, not wishful thinking.

**Key Activities**:
- Context mapping using SOCC framework (Strengths, Opportunities, Considerations, Constraints)
- Stakeholder landscape analysis
- Opportunity scoring and prioritization
- Strategic tension identification

**Outputs**:
- Shared context document
- Prioritized opportunity list with rationale
- Named strategic tensions
- Stakeholder map

**→ [See Step 1 Deep Dive](./STEP1_CONTEXT.md)**

### Step 2: DEFINE (Strategy & Plan)

**Duration**: 2-4 weeks (lightweight) to 8-12 weeks (comprehensive)

**Purpose**: Make clear choices and build coherent strategy from vision to action.

**Key Structure**: The 9-Tier Pyramid
- **Purpose** (Tiers 1-3): Vision, Values, Behaviours
- **Strategic Core** (Tiers 4-6): Drivers, Intents, Enablers
- **Execution** (Tiers 7-9): Commitments, Team Objectives, Individual Objectives

**Outputs**:
- Complete pyramid with all tiers populated
- Clear ownership from commitments down
- Red thread traceability
- Horizon-based portfolio (H1/H2/H3)

**→ [See Step 2 Deep Dive](./STEP2_STRATEGY.md)**

### Step 3: ADAPT (Living Execution)

**Duration**: Ongoing, continuous

**Purpose**: Keep strategy alive, relevant, and responsive to reality.

**Key Activities**:
- Monitor context and progress
- Learn from execution
- Adjust commitments and objectives
- Remove constraints and barriers
- Communicate changes

**Outputs**:
- Living dashboard with metrics
- Learning journal
- Adjustment log
- Regular strategy updates

**→ [See Step 3 Deep Dive](./STEP3_EXECUTION.md)**

## Key Innovations

### 1. Explicit Context Layer (Tier 0)

Most frameworks assume context or treat it as background. We make it **first-class and mandatory**.

**Innovation**: The SOCC Framework
- Structures context capture systematically
- Connects opportunities to strengths/constraints
- Provides quantitative opportunity scoring
- Surfaces strategic tensions explicitly

### 2. Red Thread Traceability

Every element requires:
- **Parent connection**: Which tier above does this support?
- **Rationale**: Why is this here? How does it connect?
- **Owner**: Who's accountable? (from commitments down)

**Innovation**: 
- Tool highlights "orphans" (disconnected elements)
- Visual traceability maps
- Forced coherence checking

### 3. Horizon Planning with Portfolio View

Strategic commitments are organized by time horizon:
- **H1 (0-12 months)**: Optimize and deliver
- **H2 (1-2 years)**: Build and scale
- **H3 (2-3 years)**: Explore and create

**Innovation**:
- Balanced portfolio view
- Warnings when over-concentrated in one horizon
- Clear sequencing and dependencies

### 4. Living Strategy System (Step 3)

Most frameworks stop at planning. We include the operating system for adaptation.

**Innovation**: Four continuous cadences
- **Monitor**: Real-time context and progress tracking
- **Learn**: Regular retrospectives and sense-making
- **Adjust**: Evidence-based modifications
- **Communicate**: Transparent updates to all stakeholders

### 5. Scalable Depth ("Onion Layers")

Same framework works for:
- Small team (10 people, lightweight version)
- Department (100 people, standard version)
- Enterprise (1000+ people, comprehensive version)

**Innovation**: Progressive disclosure
- Start simple, add depth as needed
- Core structure stays consistent
- Depth is a choice, not a requirement

## Scalability & Flexibility

### Three Depths of Implementation

#### Lightweight Strategy
**For**: Small teams, short projects, simple contexts

**Includes**:
- 1-page context (SOCC)
- Vision + 3 drivers
- 5-10 iconic commitments
- Basic team objectives
- Monthly reviews

**Time Investment**:
- Setup: 1-2 weeks
- Maintenance: 2-3 hours/month

#### Standard Strategy
**For**: Departments, products, annual planning

**Includes**:
- Full context analysis with stakeholder mapping
- Complete 9-tier pyramid
- H1/H2/H3 portfolio
- Team + individual objectives
- All four cadences (monitor, learn, adjust, communicate)

**Time Investment**:
- Setup: 6-8 weeks
- Maintenance: 4-6 hours/month

#### Comprehensive Strategy
**For**: Enterprise, transformation, multi-year planning

**Includes**:
- Deep context with scenario planning
- Full pyramid with detailed rationale
- Multiple levels (enterprise → department → team)
- Advanced metrics and dashboards
- Nested review cadences

**Time Investment**:
- Setup: 12+ weeks
- Maintenance: 8-10 hours/month

### Flexibility Principles

1. **Not all tiers required**: Start with essentials, add depth
2. **Emergent entries allowed**: Not everything must be pre-planned
3. **Multiple paths**: Can start at any step based on maturity
4. **Tool-agnostic**: Framework works with or without software

## Getting Started

### Quick Start (2 Hours)

**For teams wanting to test the approach:**

1. **Gather team** (2-8 people)
2. **Context (30 min)**: What's working? What's not? What opportunities?
3. **Vision (15 min)**: Where are we headed in 1-2 years?
4. **Drivers (30 min)**: What 3 things will we focus on?
5. **Commitments (30 min)**: What 5-8 big moves will we make?
6. **Owners (15 min)**: Who owns what?

**Output**: Lightweight strategy on a page

### Standard Start (4-6 Weeks)

**For departments/products needing robust strategy:**

**Week 1-2: Context**
- Full SOCC analysis workshop
- Stakeholder interviews
- Opportunity scoring
- Strategic tension identification

**Week 3-4: Strategy Core**
- Vision/values/behaviours definition
- Strategic drivers selection (3-5)
- Strategic intents crafting (2-3 per driver)
- Enabler identification

**Week 5-6: Execution Planning**
- Iconic commitments (H1/H2/H3)
- Team objective setting
- Individual contributions (optional)
- Red thread validation

**Output**: Complete pyramid with execution plan

### Comprehensive Start (12+ Weeks)

**For enterprise/transformation:**

Add to standard start:
- Scenario planning
- Deep stakeholder engagement
- Multi-level strategies (enterprise → department)
- Change management planning
- Communication strategy
- Dashboard development

**Output**: Full strategic architecture with living system
